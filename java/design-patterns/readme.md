
## 🎨 Design Patterns em Java

Este repositório contém implementações práticas dos principais Padrões de Projeto (Design Patterns), utilizando a linguagem Java. O objetivo é demonstrar como resolver problemas comuns de design de software de forma reutilizável, padronizada e elegante.

## 📌 O que são Design Patterns?

São soluções consolidadas para problemas recorrentes no desenvolvimento de software orientado a objetos. Eles ajudam a criar sistemas mais flexíveis, fáceis de manter e menos propensos a erros.

## 📂 Estrutura do Repositório

O projeto está dividido nas três categorias clássicas do Gang of Four (GoF):
1. Criacionais (Creational)
Focam no processo de criação de objetos, abstraindo a instância.
Singleton: Garante que uma classe tenha apenas uma instância.
Factory Method: Define uma interface para criar um objeto, mas deixa as subclasses decidirem qual classe instanciar.
Builder: Separa a construção de um objeto complexo da sua representação.
2. Estruturais (Structural)
Lidam com a composição de classes e objetos para formar estruturas maiores.
Adapter: Permite que interfaces incompatíveis trabalhem juntas.
Facade: Fornece uma interface simplificada para um sistema complexo.
Decorator: Adiciona responsabilidades a objetos dinamicamente.
3. Comportamentais (Behavioral)
Focam na comunicação e atribuição de responsabilidades entre objetos.
Strategy: Define uma família de algoritmos e os torna intercambiáveis.
Observer: Define uma dependência um-para-muitos (notificação de mudanças).
Template Method: Define o esqueleto de um algoritmo em uma operação.
