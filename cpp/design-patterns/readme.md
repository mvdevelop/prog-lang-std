
# Design Patterns em C++ 🚀

Este repositório contém implementações práticas e didáticas dos principais padrões de projeto (Design Patterns) utilizando **C++ moderno (C++17/20)**.

O objetivo é servir como um guia de referência para escrita de código reutilizável, escalável e de fácil manutenção.

## 📌 Estrutura do Repositório

Os padrões estão divididos em três categorias principais, conforme definido pela Gang of Four (GoF):

### 1. Padrões Criacionais (Creational)
Focam no processo de criação de objetos, tentando tornar o sistema independente de como seus objetos são criados.
- [ ] **Factory Method** - Criação de objetos sem especificar a classe exata.
- [ ] **Abstract Factory** - Cria famílias de objetos relacionados.
- [ ] **Builder** - Construção de objetos complexos passo a passo.
- [ ] **Singleton** - Garante uma única instância de uma classe.

### 2. Padrões Estruturais (Structural)
Lidam com a composição de classes e objetos para formar estruturas maiores.
- [ ] **Adapter** - Permite que interfaces incompatíveis trabalhem juntas.
- [ ] **Decorator** - Adiciona responsabilidades a objetos dinamicamente.
- [ ] **Facade** - Interface simplificada para um sistema complexo.
- [ ] **Proxy** - Substituto ou marcador de lugar para outro objeto.

### 3. Padrões Comportamentais (Behavioral)
Focam na comunicação entre objetos e na atribuição de responsabilidades.
- [ ] **Observer** - Notifica mudanças de estado para múltiplos objetos.
- [ ] **Strategy** - Define uma família de algoritmos intercambiáveis.
- [ ] **State** - Permite que um objeto altere seu comportamento quando seu estado interno muda.
- [ ] **Command** - Encapsula uma solicitação como um objeto.
